# QuickStock AI Instructions

## Stack
- React
- Vite
- TailwindCSS
- Firebase
- Firestore

## Architecture Principles
- Keep components modular
- Avoid over-engineering
- Prefer MVP implementations first
- Inventory items belong to households
- Recipes should reference stock items via stockItemId

## UX Philosophy
- Fast kitchen-friendly UX
- Minimize clicks/taps
- Mobile/iPad first

## Important
- Do not introduce unnecessary libraries
- Do not add backend servers unless requested
- Prefer Firestore-native solutions
- Keep Firebase free-tier friendly
